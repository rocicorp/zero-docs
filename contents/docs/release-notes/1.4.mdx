## Installation

```bash
npm install @rocicorp/zero@1.4
```

## Features

- [**Kysely Adapter:**](/docs/server-zql) Added `zeroKysely`, so Zero server mutators can run against a Kysely transaction alongside the existing Prisma and Drizzle adapters.
- [**Analyze Query Library:**](/docs/debug/slow-queries) Added `@rocicorp/zero/analyze`, making it easier to wire a project-specific `analyze-query` script into your app's schema with `runAnalyzeCLI`.
- [**Inspector Query Planner On by Default:**](/docs/debug/inspector) The inspector now always enables the query planner when analyzing queries, so join-plan output is available without extra setup.
- [**Query Names in OpenTelemetry:**](/docs/debug/otel) zero-cache now attaches custom query names to query hydration spans, making it easier to spot slow queries in your traces.
- [**Shadow Initial Sync Canary:**](/docs/zero-cache-config) zero-cache can now run an opt-in shadow initial sync against sampled upstream data to catch schema drift and full-resync failures earlier.

## Performance

- [Faster backfills by switching zero-cache to binary COPY](https://github.com/rocicorp/mono/pull/5789)
- [7.6x faster cold-start index construction for new sort orders with `BTreeSet.fromSorted`](https://github.com/rocicorp/mono/pull/5844)
- [5-16% faster query hydration by reducing `skipYields` allocations](https://github.com/rocicorp/mono/pull/5834)
- [10-15% faster query hydration and 15% faster updates](https://github.com/rocicorp/mono/pull/5801)
- [~6-8% faster query hydration for queries with joins](https://github.com/rocicorp/mono/pull/5799)
- [Avoid full table scans from reordered compound primary keys](https://github.com/rocicorp/mono/pull/5807)

## Fixes

- [Cursor pagination queries on non-nullable columns could trigger SQLite full table scans](https://github.com/rocicorp/mono/pull/5673) (thanks [@utkarsh-611](https://github.com/utkarsh-611)!)
- [Query and mutator validation errors were not exposing raw schema issues in error `details`, making custom frontend messages harder to build](https://github.com/rocicorp/mono/pull/5822) (thanks [@tjenkinson](https://github.com/tjenkinson)!)
- [Transient `litestream restore` failures were incorrectly alerting](https://github.com/rocicorp/mono/pull/5824)
- [Nullable array columns were silently converting to `[]`](https://github.com/rocicorp/mono/pull/5802) (thanks [@kvnkusch](https://github.com/kvnkusch)!)
- [Change-log entries could be incorrectly purged during handoff](https://github.com/rocicorp/mono/pull/5805)
- [Concurrent, nested, and `CREATE/DROP INDEX CONCURRENTLY` DDL statements could break schema tracking](https://github.com/rocicorp/mono/pull/5828), [follow-up](https://github.com/rocicorp/mono/pull/5829), and [follow-up](https://github.com/rocicorp/mono/pull/5858)
- [Replication slot creation could hang during startup and leave orphaned replication state behind when blocked by long-running transactions](https://github.com/rocicorp/mono/pull/5852) and [follow-up](https://github.com/rocicorp/mono/pull/5874)
- [Autoreset during startup could deadlock while recreating a replication slot](https://github.com/rocicorp/mono/pull/5876)
- [Original SQLite transaction failures could be hidden behind rollback errors](https://github.com/rocicorp/mono/pull/5862)
- [Columns added with complex Postgres defaults could be mis-translated instead of being backfilled safely](https://github.com/rocicorp/mono/pull/5868)
- [Latency histograms for hydration, lock waits, pokes, and related OTel metrics were collapsing into unusably broad buckets](https://github.com/rocicorp/mono/pull/5866)
- [Inspector parse and analyze errors were being logged like server faults instead of user-generated warnings](https://github.com/rocicorp/mono/pull/5841)
- [Stalled change-streamer transactions could hang progress indefinitely](https://github.com/rocicorp/mono/pull/5819)
- [OTel metrics from multiple worker processes could clobber each other](https://github.com/rocicorp/mono/pull/5814)
- [Drain-mode `SIGTERM` exited uncleanly during startup](https://github.com/rocicorp/mono/pull/5803)
