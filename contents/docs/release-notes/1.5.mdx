## Upgrading

### userID: "anon"

If you are using `userID: 'anon'` for logged-out users, change it to `userID: null` or `userID: undefined`. Passing `userID: 'anon'` is deprecated because it can't work with Authenticated Client Groups (below).

### Authenticated Client Groups

The `handleQueryRequest` and `handleMutateRequest` helpers now take an options object instead of multiple arguments. They also require a new `userID` parameter.

Change this:

```ts
// query handler
handleQueryRequest(
  (name, args) => {
    const query = mustGetQuery(queries, name)
    return query.fn({args})
  },
  schema,
  request,
)

// mutate handler
handleMutateRequest(
  dbProvider,
  transact => transact((tx, name, args) => {
    const mutator = mustGetMutator(mutators, name)
    return mutator.fn({args, tx})
  }),
  request,
)
```

To this:

```ts {2-3,9-10,13,15,20-21}
// query handler
handleQueryRequest({
  handler: (name, args) => {
    const query = mustGetQuery(queries, name)
    return query.fn({args})
  },
  schema,
  request,
  userID,
})

// mutate handler
handleMutateRequest({
  dbProvider,
  handler: transact => transact((tx, name, args) => {
    const mutator = mustGetMutator(mutators, name)
    return mutator.fn({args, tx})
  }),
  request,
  userID,
})
```

This is a security enhancement and is recommended for all users. See [Why pass `userID` back to Zero?](/docs/auth#server-verified-user-id) and [PR 5836](https://github.com/rocicorp/mono/pull/5836) for more information. The old signature is still supported but deprecated.

### Deploy Order

Make sure to deploy `zero-cache` before your API server. This is [existing standard practice](/docs/self-host#rolling-updates) for Zero, but it matters for this release because the 1.5 `handleQueryRequest` and `handleMutateRequest` helpers return a new response shape that `zero-cache` 1.4 cannot parse.


## Features

- [**Schema Change Hooks:**](/docs/connecting-to-postgres#schema-change-hooks) Added `zero_<shard>.update_schemas()`, so databases that do not support or allow event triggers can manually notify Zero after schema changes. The Supabase publication-change workaround is also simpler: after `ALTER PUBLICATION`, only a trailing `COMMENT ON PUBLICATION` is needed.

## Performance

- [Faster flipped-join fetches when the parent key is unique](https://github.com/rocicorp/mono/pull/5881)

## Fixes

- [`zero-cache` enforces that authenticated `userID` matches clientGroup](https://github.com/rocicorp/mono/pull/5836)
- [`CREATE/DROP INDEX CONCURRENTLY` could break replication](https://github.com/rocicorp/mono/pull/5895)
- [Long-lived connections could miss periodic auth revalidation](https://github.com/rocicorp/mono/pull/5838)
- [Allow omitting `ctx` param when no context type is defined](https://github.com/rocicorp/mono/pull/5885)
- [Make `/keepalive` timeout + graceful-shutdown opt-in except in ECSs](https://github.com/rocicorp/mono/pull/5892)
- [`Error: pipelines must be initialized` during transform or drain](https://github.com/rocicorp/mono/pull/5904)
- [Shadow sync could miss its first run before `replication-manager` restarted](https://github.com/rocicorp/mono/pull/5883)
- [Row-set drift across view-syncer re-hydration could result in stale rows](https://github.com/rocicorp/mono/pull/5857)
- [Make WebSocket compression and max-payload settings client-only](https://github.com/rocicorp/mono/pull/5919)

## Breaking Changes

None. Two APIs were deprecated:

- `userID: 'anon'` for logged-out clients
- The positional signatures of `handleQueryRequest` and `handleMutateRequest`

 See [Upgrading](#upgrading) for more information.
